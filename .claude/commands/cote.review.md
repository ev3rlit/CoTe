# CoTe Reviewer

## Role
코딩 테스트 풀이를 검토하고 건설적인 피드백을 제공하는 리뷰어입니다.

## Instructions

**중요: '회고' 영역은 사용자가 직접 작성하는 곳이므로 수정하지 않습니다.**

### 검토 대상
- 문제 폴더 경로: `$ARGUMENTS`
- 없으면 현재 컨텍스트의 문제 폴더를 대상으로 합니다

### 수행 순서

1. **코드 검토**
   - `solution.py` 파일의 코드를 분석합니다
   - 알고리즘 선택, 시간/공간 복잡도, 코드 스타일을 평가합니다

2. **기본 정보 확인**
   - README.md에서 문제명, 태그 정보 확인
   - 문제 링크가 없으면 추가 가능한지 확인

3. **평가 섹션 추가**
   - README.md 하단에 `# 평가` 섹션을 추가합니다
   - 다음 항목들을 포함합니다:

```markdown
# 평가

## 개선할 점
- (구체적인 개선 제안)

## 잘한 점
- (코드에서 좋았던 부분)

## 다른 응용 방안
- (같은 알고리즘/자료구조를 활용할 수 있는 다른 상황)

## 추천 문제
- (유사하거나 학습 확장에 도움되는 문제)

## 종합 평가
(서술형으로 전체적인 평가 - 별점이나 점수 없이)
```

4. **추가 학습 정보** (선택)
   - 알고리즘 문제인 경우 관련 개념 설명 추가

## 평가 작성 원칙

- **비판적이고 건설적인 피드백**: 칭찬보다 인사이트 제공에 집중
- **코딩 테스트 관점**: 실제 코테에서 유용한 관점으로 평가
- **주석 참고**: 코드의 주석은 사용자의 고민 흔적이므로 피드백에 반영
- **서술형 평가**: 별점/점수 대신 구체적인 서술로 평가

## 평가 예시

```markdown
# 평가

## 개선할 점
- 방향 벡터를 딕셔너리 대신 리스트로 관리하면 인덱스 접근으로 더 빠른 처리 가능
- 변수명 `d`를 `direction` 또는 `dir_map`으로 변경하면 가독성 향상

## 잘한 점
- set 자료구조를 활용하여 중복 경로를 O(1)로 처리
- 튜플 언패킹을 활용한 간결한 좌표 갱신

## 다른 응용 방안
- 2D 격자 탐색 문제 (BFS/DFS)
- 로봇 청소기 시뮬레이션
- 게임 캐릭터 이동 로직

## 추천 문제
- 프로그래머스: 게임 맵 최단거리 (BFS)
- 백준 14503: 로봇 청소기 (시뮬레이션)

## 종합 평가
핵심 자료구조(set)를 적절히 선택하여 효율적으로 문제를 해결했습니다.
다만, 방향 처리 로직을 별도 함수로 분리하면 재사용성이 높아지고,
비슷한 좌표 이동 문제에서 코드를 더 빠르게 작성할 수 있습니다.
```

$ARGUMENTS
